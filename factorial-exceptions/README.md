# Factorial exceptions
Файл Factorials.java содержит программу, которая вызывает метод класса MathUtils 
для вычисления факториала целого числа, введенного пользователем. 

Протестируйте работу программы с несколькими положительными числами, 
затем попробуйте отрицательные числа. Вы можете обнаружить, что программа 
работает для небольшого положительного целого числа (значения < 17). 

Программа возвращает большое отрицательное значение для больших целых чисел 
и всегда возвращает 1 для отрицательного целого числа.

1) Возвращение 1 в качестве факториала любого отрицательного целого числа неверно 
- математически функция факториала не определена для отрицательных чисел. 
Чтобы исправить это, вы должны изменить метод factorial, чтобы проверять, 
является ли аргумент отрицательным. 

Но есть проблема. Метод должен возвращать значение, и даже если он печатает 
сообщение об ошибке, любое возвращаемое значение может быть неверно истолкованы. 
Вместо этого он должен выдать исключение, указывающее, что что-то пошло не так, 
чтобы он не смог завершить вычисление. 

Вы можете определить свой собственный класс исключений, но уже есть исключение, 
подходящее для этой ситуации: IllegalArgumentException, 
который расширяет RuntimeException. 

Измените программу следующим образом:
*	Измените метод factorial, чтобы указать, что factorial может генерировать исключение IllegalArgumentException. 
*	Измените тело метода, чтобы проверить значение аргумента и, если оно отрицательное, бросить IllegalArgumentException в котором указывается причина возникновения ошибки. 
*	Измените метод main в  классе Factorials, чтобы перехватывать исключение, выбрасываемое factorial, и печатать соответствующий сообщение, после чего программа должна продолжить работу. 

2) Возвращать отрицательное число для значений свыше 16 также не правильно. 
Проблема в арифметическом переполнении - факториал получается больше, 
чем может быть представлено типом int. 

Это также можно рассматривать как исключение IllegalArgumentException. 
Метод определен только для аргументов до 16. Измените код в методе factorial, 
чтобы в ситуации если аргумент больше 16, также как и для отрицательного аргумента, 
метод должен бросить IllegalArgumentException, но при этом необходимо передавать различные сообщения.
